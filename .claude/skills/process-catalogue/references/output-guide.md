# Output Guide

How the final output Excel file is generated by `generate_output_excel()` in `webapp/backend/processor.py`.

---

## How Output is Triggered

Output is generated via `POST /api/export` in `console/server.py`. It:
1. Reads confirmed mappings from the frontend
2. Fetches master detail fields from `Master.csv.db` for each matched catalogue name
3. Calls `generate_output_excel()` to write the Excel file
4. Returns the file as a download

---

## Sheet Structure

The output Excel has **3 sheets**:

| Sheet | Contents |
|---|---|
| `Pathology` | Matched/confirmed rows where `dept = Pathology` |
| `Radiology` | Matched/confirmed rows where `dept = Radiology` |
| `UNMATCHED` | Rows with status `unmatched` — require manual review |

**Skipped rows** (`SKIPPED`) are excluded entirely — not written to any sheet.

## Provider Slug Rule

`provider_item_name` is always derived from the **Standard Name (catalogue name)**:

```python
re.sub(r'[^a-z0-9]+', '-', catalogue_name.strip().lower()).strip('-')
```

- Never derived from the raw provider test name
- Applied in `_val()` inside `generate_output_excel()` and in `_build_row()` in `server.py`

---

## Row Inclusion Rules

- **Matched sheet**: rows with `status` = `matched` or `confirmed` AND a non-empty `standard_name`
- **UNMATCHED sheet**: rows with `status` = `unmatched`
- **Excluded**: rows with `status` = `skipped` or `rejected`

---

## Output File Path

```
output/<provider_name>_standardized_catalogue.xlsx
```

Provider name is inferred from the input filename. File is saved under `output/` in the project root.
